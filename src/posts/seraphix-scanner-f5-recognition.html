I built a tool called Seraphix. It's a secret scanning toolkit designed for bug bounty hunters and security researchers who need to scan GitHub at scale. The kind of scale where you're not just checking one repo - you're auditing entire organizations, hunting through force-pushed commits, and finding secrets people thought they deleted.

What makes it different

Most scanners are built for single-org audits and hit API rate limits fast. Seraphix is built for mass scanning - it can hammer through hundreds of orgs and repos in parallel, all running locally to avoid being throttled by GitHub's API. When you need to scan 50+ organizations at once, this matters.

Plus, it goes deeper. When developers accidentally commit secrets and panic with a git push --force to "delete" them, those commits often become dangling but still accessible. Seraphix specifically targets these force-pushed commits using filtered GHArchive data.

Seraphix hunting for secrets in a force-pushed commit

The idea came from Sharon Brizinov's work on scanning GitHub's "oops" commits. I wanted that capability in a toolkit I could actually use for real hunting.

Core features

Mass organization scanning - Point it at a GitHub org (or a list of orgs) and it'll parallelize everything automatically. Calculates optimal workers based on your system.

Force-push goldmine - The unique advantage. Finds secrets in commits people tried to erase. Uses TruffleHog for verification.

Bug bounty utility - Generates lists of GitHub organizations with bug bounty programs. Feed that directly into the mass scanner.

Targeted scanning - Can also focus on specific repos or individual commits when you need precision.

Analytics that actually help

Every scan generates an interactive dashboard showing secret statistics, risk assessment, and verification status. You can enable real-time alerts via Telegram, Discord, or email so you don't have to wait for scans to finish.

Interactive dashboard for analyzing discovered secrets

The result

Using Seraphix led to me discovering vulnerabilities that got me acknowledged by F5 in their SIRT Security Researcher Acknowledgement program. Pretty cool seeing your name there.

Quick start

git clone https://github.com/AmedeoV/seraphix.git
cd seraphix
./install_requirements.sh

# Scan an org (e.g., top 10 Microsoft repos)
cd org-scanner/
./scan_org.sh microsoft --max-repos 10

It's written in Shell and Python, so it's portable and easy to deploy.
